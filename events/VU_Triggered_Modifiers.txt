####													  ####
####    Triggered Modifiers performance upgrade by Don    ####
####													  ####
##############################################################
#############################################################################################################################################################################################
# 1)
#	It may be somewhat more complicated than used to, but it simplifies the options a lot, meaning less code, meaning smaller mess, it should be more convenient than doing it manually
#	and it was made this way so that you can translate the triggered modifiers into this format as easily as possible.
#
#	Do keep in mind that there's no more 'Potential', and therefore you have to merge it with the 'Trigger'.
#
#############################################################################################################################################################################################
# 2)
#	Use the vu_triggered_modifier_trigger scripted trigger for the trigger for the events, it needs a modifier and a trigger, please ignore the "error" of the trigger, it's a false flag.
#	It should be able to handle most things on it's own meaning that you don't have to do much, but there are situations that require a bit more effort on your part to make work.
#	add the trigger into "" marks. 
#
#############################################################################################################################################################################################
# 3)
#	If you get an error saying "Quoted string longer than max length (512)", you just need to copy the contents of the vu_triggered_modifier_trigger into a new scripted trigger, which
#	may be in turn used inside vu_triggered_modifier_trigger once more, going around the 512 character limit. This solution should be prefered to 4) in all situation where possible.
#
#	As an example and template in this situation use: vu_triggered_modifier_event_57_trigger
#
#############################################################################################################################################################################################
# 4)
#	For instances that require "" marks inside the trigger (like dynasty = "von Wittelsbach"), you need to copy the body of vu_triggered_modifier_trigger into a brand new scripted trigger, 
#	which you should name accordingly (example below) Then you need replace the $trigger$ with your trigger, and $modifier$ with the modifier you desire. 
#
#	This also may be used when the game's acting up and being pissy about some triggers not validating, causing the 'failed trigger validation' error. (usually caused by institutions)
#
#	I'm sorry it's a bit more complicated when this comes about, but PDX isn't the best at making functional programming languages, and therefore  many things that should logically work 
#	(like this and the 512 error), don't.
#
#	For example and template use: vu_triggered_modifier_event_103_trigger
#
#############################################################################################################################################################################################
# 5)
#	For the option, just re-use the same modifier you used in vu_triggered_modifier_trigger, and add it into the vu_triggered_modifier_effect scripted effect, it handles the switching
#	on it's own, no changes or nothing needed. If you come to any issues using it, please don't touch the thing itself, and inform me (Don) posthaste about the issue, so we can solve it.
#
#############################################################################################################################################################################################
# 6)
#	Notes to specific events:
#
#	57, 58, 59, 60, 71 & 110 all use the 3) system, as they were too long to fit within the vu_triggered_modifier_trigger's 512 character limit
#	6, 7, 103 & 104 all use the 4) system as the dynasties use "" marks
#	20 21 & 43 use the 4) system as institutions cause 'failed trigger validation' errors in the error.log if done in the 2) way
#
#############################################################################################################################################################################################
#
#

		if = {
			limit = {
				has_country_modifier = VU_maritime_republic_triggered_modifier_new_world
			}
			NOT = {
				AND = {
					has_government_mechanic = maritime_republic_mechanic
					is_institution_enabled = new_world_i
				}
			}
		}
		else_if = {
			limit = {
				NOT = {
					has_country_modifier = VU_maritime_republic_triggered_modifier_new_world
				}
			}

			is_institution_enabled = new_world_i
		}
	}
	immediate = {
		vu_triggered_modifier_effect_desc = {
			modifier = VU_maritime_republic_triggered_modifier_new_world
		}
